## 🚀 部署步骤

### 步骤 1：在 Vercel 中部署项目

1. **访问 Vercel**
   - 打开浏览器，访问 https://vercel.com
   - 使用 GitHub 账号登录

2. **导入项目**
   - 点击 "New Project" 按钮
   - 在项目列表中找到 `shongting/y-car-website`
   - 点击 "Import" 按钮

3. **配置项目**
   - Vercel 会自动检测到 `vercel.json` 配置
   - 无需修改任何设置
   - 直接点击 "Deploy" 按钮

4. **等待部署完成**
   - 部署通常需要 1-2 分钟
   - 可以查看实时部署日志
   - 部署成功后会显示 "Congratulations!" 页面

5. **获取 URL**
   - 复制生产环境 URL
   - 格式通常为：`https://y-car-website.vercel.app` 或 `https://y-car-website-[hash].vercel.app`

## 🔍 验证步骤

### 方法 1：使用自动化测试工具（推荐）

1. **打开测试工具**
   ```bash
   # 在浏览器中打开
   open y-car-website/deployment-test.html
   ```

2. **输入部署 URL**
   - 将从 Vercel 获取的 URL 粘贴到输入框
   - 点击"开始验证"按钮

3. **查看测试结果**
   - 工具会自动运行 8 项测试
   - 查看每项测试的通过/失败状态
   - 记录测试结果

### 方法 2：使用命令行脚本

```bash
# 运行验证脚本
./y-car-website/check-deployment.sh https://your-site.vercel.app

# 替换为您的实际 URL
```

脚本会检查：
- ✅ HTTPS 连接
- ✅ 网站可访问性
- ✅ 响应时间
- ✅ 内容类型
- ✅ 页面大小
- ✅ 关键内容

### 方法 3：手动验证（必需）

#### 桌面端验证

1. **在浏览器中打开部署 URL**
   - 使用 Chrome、Firefox 或 Safari

2. **验证 HTTPS 连接**
   - 检查地址栏是否显示锁图标 🔒
   - 点击锁图标查看证书信息

3. **验证页面加载**
   - 打开开发者工具（F12）
   - 切换到 Network 标签
   - 刷新页面
   - 检查加载时间是否 < 3 秒

4. **验证页面内容**
   - [ ] Header 导航栏显示正常
   - [ ] Hero 区域显示正常
   - [ ] 车型卡片显示正常（3 列布局）
   - [ ] 车型图片加载正常
   - [ ] 新能源优势区块显示正常（3 列）
   - [ ] 联系表单显示正常
   - [ ] Footer 显示正常

5. **验证交互功能**
   - [ ] 点击导航链接，页面平滑滚动
   - [ ] 点击车型图片，显示放大模态框
   - [ ] 点击模态框背景或关闭按钮，模态框关闭
   - [ ] 悬停按钮和卡片，显示悬停效果
   - [ ] 点击"立即咨询"按钮，显示联系表单

6. **验证表单功能**
   - [ ] 输入无效数据，显示错误提示
   - [ ] 输入有效数据，提交成功
   - [ ] 提交后显示成功消息
   - [ ] 表单字段被清空
   - [ ] 数据保存到 LocalStorage

#### 移动端验证

1. **使用浏览器开发者工具**
   - 按 F12 打开开发者工具

## 🔧 故障排除

### 问题 1：部署失败

**可能原因**：
- vercel.json 配置错误
- 文件路径不正确
- GitHub 仓库权限问题

**解决方案**：
1. 检查 vercel.json 配置
2. 查看 Vercel 部署日志
3. 确认 GitHub 仓库可访问

### 问题 2：页面显示异常

**可能原因**：
- CSS 或 JS 文件路径错误
- 资源加载失败
- 浏览器缓存问题

**解决方案**：
1. 打开浏览器控制台查看错误
2. 检查 Network 标签确认资源加载
3. 清除浏览器缓存并刷新

### 问题 3：图片加载失败

**可能原因**：
- 图片 URL 不可访问
- CORS 问题
- 网络问题

**解决方案**：
1. 检查图片 URL 是否有效
2. 确认占位符图片正常显示
3. 查看控制台错误信息

### 问题 4：表单提交失败

**可能原因**：
- JavaScript 错误
- LocalStorage 不可用
- 表单验证问题

**解决方案**：
1. 打开控制台查看 JavaScript 错误
2. 检查浏览器是否支持 LocalStorage
3. 测试表单验证逻辑
